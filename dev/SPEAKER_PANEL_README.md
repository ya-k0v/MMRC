# 🎤 Спикер панель MMRC

## Описание

Спикер панель — интерфейс для управления воспроизведением контента на устройствах в реальном времени. Предназначена для спикеров и операторов, которые управляют контентом во время презентаций или мероприятий.

**URL:** `/speaker.html`  
**Доступ:** Требуется роль `admin` или `speaker`  
**Файл:** `public/speaker.html`

---

## Назначение

Спикер панель позволяет:
- **Выбирать устройство** для управления
- **Выбирать файл** для воспроизведения
- **Управлять воспроизведением** (play, pause, stop, restart)
- **Переключать страницы** PDF/PPTX презентаций
- **Управлять громкостью** устройств
- **Просматривать превью** файлов перед воспроизведением

---

## Структура панели

### 1. Боковая панель (Sidebar)

**Расположение:** Слева (на широких экранах) или сверху (на мобильных)

**Элементы:**
- **Список устройств** (`tvList`) — все доступные устройства
- **Индикатор громкости** — текущая громкость выбранного устройства
- **Панель управления громкостью** — ползунок и кнопка mute/unmute

**Отображение устройств:**
- Имя устройства
- ID устройства
- Статус онлайн/офлайн (🟢/⚪)
- Текущий воспроизводимый файл
- Индикатор громкости

### 2. Основная область (Main)

**Расположение:** Центр (на широких экранах) или снизу (на мобильных)

**Элементы:**
- **Список файлов** (`fileList`) — файлы выбранного устройства
- **Превью файла** (`filePreview`) — предпросмотр выбранного файла

### 3. Панель управления громкостью (Volume Panel)

**Расположение:** Справа (на широких экранах) или внизу (на мобильных)

**Элементы:**
- **Ползунок громкости** — изменение уровня (0-100%, шаг 5%)
- **Индикатор уровня** — текстовое отображение (например: "75%")
- **Кнопка mute/unmute** — включение/выключение звука
- **Статус звука** — "Звук включен" / "Звук выключен" / "Устройство офлайн"

## Функциональность

### Выбор устройства

**Способы:**
1. Кликнуть на устройство в списке устройств (боковая панель)
2. Устройство автоматически выбирается при первом подключении

**Что происходит:**
- Загружается список файлов устройства
- Загружается состояние громкости
- Обновляется превью (если есть выбранный файл)

### Выбор файла для воспроизведения

**Способы:**
1. Кликнуть на файл в списке файлов
2. Файл автоматически выбирается при открытии устройства

**Отображение файлов:**
- Имя файла
- Размер файла
- Тип контента (видео, изображение, PDF, PPTX, папка)
- Статус обработки (ready, processing, error)

**Превью:**
- **Изображения** — показывается миниатюра
- **Видео** — показывается первый кадр (если доступен трейлер)
- **PDF/PPTX** — показывается первая страница/слайд

### Управление воспроизведением

**Команды через Socket.IO:**

#### ▶️ Воспроизвести (Play)
- **Действие:** Начать воспроизведение выбранного файла
- **Команда:** `player/play`
- **Параметры:** `{ device_id, file, type }`
- **Результат:** Устройство начинает воспроизведение

#### ⏸️ Пауза (Pause)
- **Действие:** Приостановить воспроизведение с сохранением позиции
- **Команда:** `player/pause`
- **Параметры:** `{ device_id }`
- **Результат:** Воспроизведение приостанавливается

#### ⏹️ Остановка (Stop)
- **Действие:** Остановить воспроизведение и вернуться к заглушке
- **Команда:** `player/stop`
- **Параметры:** `{ device_id }`
- **Результат:** Воспроизведение останавливается, показывается заглушка

#### 🔄 Перезапуск (Restart)
- **Действие:** Перезапустить воспроизведение с начала
- **Команда:** `player/restart`
- **Параметры:** `{ device_id }`
- **Результат:** Воспроизведение начинается с начала

#### ⏩ Перемотка (Seek)
- **Действие:** Перемотать на определённую позицию
- **Команда:** `player/seek`
- **Параметры:** `{ device_id, position }` (в секундах)
- **Результат:** Воспроизведение перематывается на указанную позицию

### Управление PDF/PPTX презентациями

**Навигация по страницам/слайдам:**

#### ◀️ Предыдущая страница
- **Кнопка:** "◀ Предыдущая" в превью
- **Действие:** Переключение на предыдущую страницу/слайд
- **Команда:** `player/folderPage`
- **Параметры:** `{ device_id, page: currentPage - 1 }`

#### ▶️ Следующая страница
- **Кнопка:** "Следующая ▶" в превью
- **Действие:** Переключение на следующую страницу/слайд
- **Команда:** `player/folderPage`
- **Параметры:** `{ device_id, page: currentPage + 1 }`

**Особенности:**
- Страницы нумеруются с 1
- Автоматическая проверка границ (нельзя перейти за пределы)
- Синхронизация между всеми подключёнными клиентами
- Подсветка текущей страницы в превью

### Управление папками с изображениями

**Папки** — это ZIP архивы с изображениями, которые воспроизводятся как слайд-шоу.

**Настройки:**
- **Интервал переключения:** 5, 10, 15, 20 секунд (по умолчанию 10)
- **Режим:** Автоматическое переключение или ручное

**Управление:**
- Переключение страниц работает так же, как для PDF/PPTX
- Можно установить интервал переключения через админ панель

### Управление громкостью

**Элементы управления:**
- **Ползунок** — изменение уровня громкости (0-100%)
- **Индикатор уровня** — текстовое отображение (например: "75%")
- **Кнопка mute/unmute** — включение/выключение звука

**Особенности:**
- Шаг изменения: 5%
- Изменения применяются мгновенно через Socket.IO
- Состояние сохраняется в БД
- Синхронизируется между всеми подключёнными клиентами

**Команды:**
- `POST /api/devices/:id/volume` — установить громкость
- Socket.IO событие `player/volume` — мгновенное обновление

**Статусы:**
- 🟢 **Звук включен** — устройство онлайн, звук включён
- 🔴 **Звук выключен** — устройство онлайн, звук выключен
- ⚪ **Устройство офлайн** — устройство не подключено

### Превью файлов

**Поддерживаемые типы:**

#### Изображения
- Показывается миниатюра в списке файлов
- Полноразмерное превью при клике
- Поддержка: PNG, JPG, JPEG, GIF, WEBP

#### Видео
- Показывается первый кадр (если доступен трейлер)
- Видео плеер при клике
- Поддержка: MP4, WEBM, OGG, MKV, MOV, AVI

#### PDF/PPTX
- Показывается первая страница/слайд
- Навигация по страницам/слайдам
- Автоматическая конвертация в PNG изображения

#### Папки
- Показывается первое изображение из папки
- Навигация по изображениям
- Автоматическое извлечение из ZIP архива

## Технические детали

### Модули JavaScript

**Основной файл:** `public/js/speaker.js`

**Импорты:**
- `theme.js` — переключение темы
- `utils.js` — утилиты (сортировка, debounce, пагинация)
- `speaker/auth.js` — аутентификация
- `shared/svg-icons.js` — SVG иконки
- `shared/formatters.js` — форматирование времени

**Основные функции:**
- `loadDevices()` — загрузка списка устройств
- `loadFiles()` — загрузка списка файлов
- `previewFile()` — превью файла
- `sendPlayCommand()` — команда воспроизведения
- `sendPauseCommand()` — команда паузы
- `sendStopCommand()` — команда остановки
- `sendRestartCommand()` — команда перезапуска
- `sendFolderPageCommand()` — переключение страницы папки
- `updateVolumeUI()` — обновление UI громкости
- `sendVolumeCommand()` — команда изменения громкости

### Socket.IO события

**Подписка на события:**
- `devices/updated` — обновление списка устройств
- `players/onlineSnapshot` — снимок онлайн устройств
- `player/online` — устройство подключилось
- `player/offline` — устройство отключилось
- `devices/volume/state` — обновление громкости
- `devices/volume/stateBatch` — пакетное обновление громкости

**Отправка событий:**
- `player/play` — воспроизвести
- `player/pause` — пауза
- `player/stop` — остановка
- `player/restart` — перезапуск
- `player/seek` — перемотка
- `player/folderPage` — переключение страницы папки
- `player/volume` — управление громкостью

### API Endpoints

**Используемые endpoints:**
- `GET /api/devices` — список устройств
- `GET /api/devices/:id/files-with-status` — список файлов
- `GET /api/files/resolve/:deviceId/:fileName` — получение файла для превью
- `GET /api/devices/:id/volume` — получить громкость
- `POST /api/devices/:id/volume` — установить громкость

### PWA (Progressive Web App)

**Особенности:**
- Service Worker для офлайн работы
- Автоматическая очистка кэша при запуске
- Манифест: `manifest-speaker.json`
- Иконки: `icon-192.png`, `icon-512.png`

**Очистка кэша:**
- При каждом запуске проверяется флаг `vc-speaker-cache-cleared-v1`
- Если флаг не установлен — очищается кэш Service Worker
- Страница перезагружается для обновления ресурсов

### Адаптивность

**Мобильные устройства (< 1024px):**
- Вертикальная компоновка
- Боковая панель сверху
- Основная область снизу
- Панель громкости внизу

**Десктоп (≥ 1024px):**
- Горизонтальная компоновка
- Боковая панель слева (280px)
- Основная область в центре
- Панель громкости справа (360px)

## Сценарии использования

### Сценарий 1: Презентация с PDF

1. **Выбор устройства:** Кликнуть на устройство в списке
2. **Выбор файла:** Кликнуть на PDF файл
3. **Просмотр превью:** Проверить первую страницу
4. **Воспроизведение:** Нажать кнопку "Воспроизвести"
5. **Навигация:** Использовать кнопки "◀ Предыдущая" / "Следующая ▶" для переключения страниц

### Сценарий 2: Видео презентация

1. **Выбор устройства:** Кликнуть на устройство в списке
2. **Выбор файла:** Кликнуть на видео файл
3. **Просмотр превью:** Проверить трейлер (первые 10 секунд)
4. **Воспроизведение:** Нажать кнопку "Воспроизвести"
5. **Управление:** Использовать кнопки "Пауза", "Остановка", "Перезапуск"

### Сценарий 3: Слайд-шоу из папки

1. **Выбор устройства:** Кликнуть на устройство в списке
2. **Выбор папки:** Кликнуть на папку (ZIP архив с изображениями)
3. **Просмотр превью:** Проверить первое изображение
4. **Воспроизведение:** Нажать кнопку "Воспроизвести"
5. **Навигация:** Использовать кнопки для переключения изображений

### Сценарий 4: Управление громкостью

1. **Выбор устройства:** Кликнуть на устройство в списке
2. **Изменение громкости:** Использовать ползунок (шаг 5%)
3. **Включение/выключение звука:** Нажать кнопку mute/unmute
4. **Проверка статуса:** Проверить индикатор статуса звука

## Советы по использованию

1. **Подготовка перед мероприятием:**
   - Проверьте, что все файлы загружены и обработаны (статус `ready`)
   - Проверьте онлайн статус устройств
   - Установите нужную громкость заранее

2. **Во время мероприятия:**
   - Используйте превью для проверки файлов перед воспроизведением
   - Следите за статусом устройств (онлайн/офлайн)
   - Используйте кнопку "Перезапуск" для повторного воспроизведения

3. **Управление громкостью:**
   - Изменения применяются мгновенно
   - Состояние сохраняется между сессиями
   - Можно управлять громкостью с нескольких устройств одновременно

4. **Навигация по презентациям:**
   - Страницы нумеруются с 1
   - Автоматическая проверка границ
   - Синхронизация между всеми клиентами

## Решение проблем

### Устройство не отображается онлайн

**Проверьте:**
1. Устройство подключено к сети
2. Socket.IO соединение установлено
3. Device ID совпадает с настройками устройства
4. Обновите страницу (F5)

### Файл не воспроизводится

**Проверьте:**
1. Статус файла (должен быть `ready`)
2. Формат файла поддерживается устройством
3. Файл не повреждён
4. Устройство онлайн

### Громкость не изменяется

**Проверьте:**
1. Устройство онлайн
2. Socket.IO соединение установлено
3. Обновите страницу (F5)

### Превью не загружается

**Проверьте:**
1. Файл существует на сервере
2. Права доступа к файлу
3. Формат файла поддерживается
4. Обновите страницу (F5)
