# 🎖️ Hero Module — Картотека героев

## Описание

Hero Module — модуль для управления картотекой героев (биографиями). Полностью изолирован в отдельной структуре с собственной базой данных и интерфейсами.

**Публичная панель URL:** `/hero/index.html` или `/hero`  
**Админ-панель URL:** `/hero/admin.html`  
**Доступ:** Публичная панель — без авторизации, админ-панель — требует роль `admin` или `hero_admin`

---

## Назначение

Hero Module предназначен для:
- **Хранения биографий** героев (текстовые описания)
- **Управления медиаматериалами** (фото, видео) для каждого героя
- **Поиска героев** по ФИО
- **Отображения карточек** героев на публичных экранах

---

## Структура модуля

### Клиентский код (Frontend)

**Публичная панель:**
- `public/hero/index.html` — публичная панель героев (поиск и отображение)
- `public/hero/js/hero.js` — клиентский код для публичной панели
- `public/hero/js/hero-utils.js` — общие утилиты

**Админ-панель:**
- `public/hero/admin.html` — админ-панель героев (CRUD операции)
- `public/hero/js/hero-admin.js` — клиентский код для админ-панели
- `public/hero/js/hero-utils.js` — общие утилиты

**Стили:**
- `public/css/hero.css` — стили для панелей героев

### Серверный код (Backend)

- `src/hero/index.js` — главный экспорт модуля (создание роутера)
- `src/hero/routes/hero-router.js` — API роуты для героев
- `src/hero/database/hero-db.js` — инициализация базы данных
- `src/hero/database/queries.js` — запросы к базе данных
- `src/hero/database/schema.sql` — схема базы данных

### База данных

- `config/hero/heroes.db` — SQLite база данных героев (создается автоматически при первом запуске)

**Схема БД:**
- `heroes` — таблица героев (id, full_name, biography, photo_url, video_url, created_at, updated_at)
- Автоматические миграции при старте сервера

## Публичная панель (`/hero/index.html`)

### Назначение

Публичная панель предназначена для поиска и отображения карточек героев на экранах (киосках, дисплеях).

### Структура

**Элементы:**
- **Поисковая строка** — ввод ФИО для поиска
- **Выпадающий список** — подсказки при вводе
- **Область контента** — отображение карточки героя
- **Пустое состояние** — сообщение при отсутствии поиска

### Функциональность

#### Поиск героев

**Способы:**
1. Ввести ФИО в поисковую строку
2. Выбрать героя из выпадающего списка подсказок

**Особенности поиска:**
- Поиск по началу строки (prefix search)
- Регистронезависимый поиск
- Нормализация 'е' и 'ё' (например: "Елена" найдёт "Елена" и "Ёлена")
- Автоматические подсказки при вводе

**API:**
- `GET /api/hero/search?q=query` — поиск героев

#### Отображение карточки героя

**Элементы карточки:**
- **ФИО** — полное имя героя
- **Биография** — текстовое описание (книжное оформление)
- **Фото** — изображение героя (если есть)
- **Видео** — видео материал (если есть)

**Оформление:**
- Книжное оформление биографии (красная строка, выравнивание по ширине)
- Адаптивный дизайн для разных размеров экранов
- Lightbox для просмотра фото/видео в полном размере

### Технические детали

**Модули:**
- `hero.js` — основной код публичной панели
- `hero-utils.js` — утилиты (поиск, нормализация, отображение)

**API Endpoints:**
- `GET /api/hero/search?q=query` — поиск героев
- `GET /api/hero/:id` — получить героя по ID

## Админ-панель (`/hero/admin.html`)

### Назначение

Админ-панель предназначена для управления картотекой героев (создание, редактирование, удаление).

### Структура

**Верхняя панель (Toolbar):**
- **ФИО пользователя** — отображается текущий пользователь
- **Кнопка "Выйти"** — выход из системы
- **Кнопка "Экспорт БД"** — экспорт базы данных героев
- **Кнопка "Добавить"** — создание нового героя

**Основная область:**
- **Список героев** (слева) — все герои в системе
- **Детальная информация** (справа) — редактирование выбранного героя

### Функциональность

#### Просмотр списка героев

**Элементы:**
- **Счетчик героев** — общее количество героев
- **Поисковая строка** — фильтрация по ФИО
- **Список героев** — карточки всех героев

**Отображение:**
- ФИО героя
- Дата создания/обновления
- Индикатор наличия фото/видео

#### Создание героя

**Шаги:**
1. Нажать кнопку **"Добавить"** в toolbar
2. В форме ввести:
   - **ФИО** — полное имя героя (обязательно)
   - **Биография** — текстовое описание (опционально)
   - **Фото URL** — ссылка на изображение (опционально)
   - **Видео URL** — ссылка на видео (опционально)
3. Нажать **"Сохранить"**

**API:**
- `POST /api/hero/` — создать героя

#### Редактирование героя

**Шаги:**
1. Выбрать героя из списка
2. В форме редактирования изменить данные
3. Нажать **"Сохранить"**

**API:**
- `PUT /api/hero/:id` — обновить героя

#### Удаление героя

**Шаги:**
1. Выбрать героя из списка
2. Нажать кнопку **"Удалить"** в форме редактирования
3. Подтвердить удаление

**API:**
- `DELETE /api/hero/:id` — удалить героя

#### Экспорт базы данных

**Назначение:** Создание резервной копии базы данных героев.

**Шаги:**
1. Нажать кнопку **"Экспорт БД"** в toolbar
2. Файл `heroes.db` будет скачан

**API:**
- `GET /api/hero/export-database` — экспорт базы данных

### Технические детали

**Модули:**
- `hero-admin.js` — основной код админ-панели
- `hero-utils.js` — утилиты (поиск, нормализация, валидация)

**API Endpoints:**
- `GET /api/hero/` — получить всех героев
- `GET /api/hero/search?q=query` — поиск героев
- `GET /api/hero/:id` — получить героя по ID
- `POST /api/hero/` — создать героя
- `PUT /api/hero/:id` — обновить героя
- `DELETE /api/hero/:id` — удалить героя
- `GET /api/hero/export-database` — экспорт БД

**Аутентификация:**
- Требуется роль `admin` или `hero_admin`
- JWT токены (access 12h, refresh 30d)

## API Endpoints

### Публичные endpoints (без авторизации)

```
GET /api/hero/search?q=query
  Query: q — поисковый запрос (ФИО)
  Response: [{ id, full_name, biography, photo_url, video_url, ... }]

GET /api/hero/:id
  Response: { id, full_name, biography, photo_url, video_url, created_at, updated_at }
```

### Защищённые endpoints (требуют авторизацию)

```
GET /api/hero/
  Headers: Authorization: Bearer <token> (admin или hero_admin)
  Response: [{ id, full_name, biography, photo_url, video_url, ... }]

POST /api/hero/
  Headers: Authorization: Bearer <token> (admin или hero_admin)
  Body: { full_name, biography?, photo_url?, video_url? }
  Response: { id, full_name, biography, ... }

PUT /api/hero/:id
  Headers: Authorization: Bearer <token> (admin или hero_admin)
  Body: { full_name?, biography?, photo_url?, video_url? }
  Response: { id, full_name, biography, ... }

DELETE /api/hero/:id
  Headers: Authorization: Bearer <token> (admin или hero_admin)
  Response: { ok: true }

GET /api/hero/export-database
  Headers: Authorization: Bearer <token> (admin или hero_admin)
  Response: File download (heroes.db)
```

## Особенности

### Поиск

- **Поиск по началу строки** (prefix search) — быстрый поиск по первым символам
- **Регистронезависимый** — "ИВАНОВ" найдёт "Иванов"
- **Нормализация 'е' и 'ё'** — "Елена" найдёт "Елена" и "Ёлена"
- **Автоматические подсказки** — выпадающий список при вводе

### Оформление биографий

- **Книжное оформление:**
  - Красная строка (отступ первой строки)
  - Выравнивание по ширине
  - Читабельный шрифт
- **Адаптивный дизайн** — корректное отображение на разных экранах

### Медиаматериалы

- **Фото** — поддержка изображений (URL или путь к файлу)
- **Видео** — поддержка видео (URL или путь к файлу)
- **Lightbox** — просмотр фото/видео в полном размере

### База данных

- **Автоматическое создание** — БД создаётся при первом запуске
- **Автоматические миграции** — схема обновляется при старте сервера
- **Экспорт/импорт** — возможность создания резервных копий

## Сценарии использования

### Сценарий 1: Добавление нового героя

1. Войти в админ-панель (`/hero/admin.html`)
2. Нажать кнопку **"Добавить"**
3. Заполнить форму:
   - ФИО: "Иванов Иван Иванович"
   - Биография: "Родился в 1950 году..."
   - Фото URL: `/content/heroes/ivanov.jpg`
4. Нажать **"Сохранить"**

### Сценарий 2: Поиск героя на публичном экране

1. Открыть публичную панель (`/hero/index.html`)
2. Ввести ФИО в поисковую строку: "Иванов"
3. Выбрать героя из списка подсказок
4. Просмотреть карточку героя

### Сценарий 3: Редактирование биографии

1. Войти в админ-панель
2. Выбрать героя из списка
3. Изменить биографию в форме редактирования
4. Нажать **"Сохранить"**

### Сценарий 4: Экспорт базы данных

1. Войти в админ-панель
2. Нажать кнопку **"Экспорт БД"**
3. Файл `heroes.db` будет скачан

## Решение проблем

### Герой не находится при поиске

**Проверьте:**
1. ФИО введено правильно
2. Герой существует в базе данных
3. Обновите страницу (F5)

### Не удаётся сохранить героя

**Проверьте:**
1. ФИО заполнено (обязательное поле)
2. Вы авторизованы как `admin` или `hero_admin`
3. Токен не истёк

### База данных не создаётся

**Проверьте:**
1. Права доступа к папке `config/hero/`
2. Логи сервера на наличие ошибок
3. Перезапустите сервер
