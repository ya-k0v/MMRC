# 📊 Админ панель MMRC 3.2.1

## Описание

Админ панель — основной интерфейс для управления системой MMRC. Предназначена для администраторов и позволяет полностью управлять устройствами, файлами, пользователями, стримами и настройками системы.

**URL:** `/admin.html`  
**Доступ:** Требуется роль `admin` или `speaker`  
**Файл:** `public/admin.html`

---

## Структура панели

### 1. Верхняя панель (Toolbar)

**Элементы:**
- **Переключение темы** — светлая/тёмная тема
- **ФИО пользователя** — отображается текущий пользователь
- **Кнопка "Выйти"** — выход из системы
- **Кнопка "Устройства"** — управление устройствами (только для admin)
- **Кнопка "Пользователи"** — управление пользователями (только для admin)
- **Кнопка "Картотека"** — переход в Hero модуль
- **Кнопка "Спикер"** — открытие спикер панели в новой вкладке
- **Кнопка "⚙️ Настройки"** — настройки системы

### 2. Основная область (Grid)

**Структура:**
- **Список устройств** (слева) — карточки всех устройств в системе
- **Детальная информация** (справа) — информация о выбранном устройстве
- **Панель файлов** (внизу) — список файлов выбранного устройства

## Функциональность

### Управление устройствами

#### Просмотр устройств
- **Список устройств** отображается в виде карточек
- Каждая карточка показывает:
  - Имя устройства
  - ID устройства
  - IP адрес (если доступен)
  - Статус онлайн/офлайн (зелёный/серый индикатор)
  - Текущий воспроизводимый файл
  - Громкость (иконка и уровень)
- **Пагинация** — устройства отображаются постранично

#### Создание устройства (только admin)
1. Нажать кнопку **"Устройства"** в toolbar
2. В модальном окне нажать **"Создать устройство"**
3. Ввести:
   - **Device ID** — уникальный идентификатор (например: `ATV001`)
   - **Имя** — отображаемое имя (опционально)
4. Нажать **"Создать"**

#### Переименование устройства (только admin)
1. Открыть устройство
2. В детальной информации нажать **"Переименовать"**
3. Ввести новое имя
4. Нажать **"Сохранить"**

#### Удаление устройства (только admin)
1. Нажать кнопку **"Устройства"** в toolbar
2. Выбрать устройство в списке
3. Нажать **"Удалить"**
4. Подтвердить удаление

**⚠️ Внимание:** При удалении устройства удаляются все его файлы и метаданные из БД!

### Управление файлами

#### Загрузка файлов

**Поддерживаемые форматы:**
- **Видео:** `.mp4`, `.webm`, `.ogg`, `.mkv`, `.mov`, `.avi`
- **Аудио:** `.mp3`, `.wav`, `.m4a`
- **Изображения:** `.png`, `.jpg`, `.jpeg`, `.gif`, `.webp`
- **Документы:** `.pdf`, `.pptx`
- **Архивы:** `.zip` (папки с изображениями)

**Максимальный размер:** 5 GB на файл

**Способы загрузки:**

1. **Drag & Drop:**
   - Перетащить файлы на карточку устройства
   - Файлы автоматически начнут загружаться

2. **Кнопка загрузки:**
   - Открыть устройство
   - Нажать **"Загрузить файлы"** в панели файлов
   - Выбрать файлы в диалоге

**Процесс загрузки:**
1. Файл сохраняется во временную папку
2. MIME-валидация через `file-type`
3. Транслитерация имени файла (кириллица → латиница)
4. Вычисляется MD5 хэш (частичный MD5 для файлов >100MB)
5. Проверяется наличие дубликатов
6. **MP4 Faststart** — обработка moov atom для поддержки seek в ExoPlayer
7. Файл сохраняется в папку устройства
8. Метаданные сохраняются в БД
9. Автоматическая обработка:
   - **PDF/PPTX** → конвертация в PNG изображения
   - **Видео** → проверка и оптимизация (если нужно)
   - **Трейлер** — фоновая генерация ~10-сек превью
10. Socket.IO событие `file/ready`

**Индикация:**
- Прогресс-бар показывает процент загрузки
- Статус файла: `pending` → `checking` → `processing` → `ready` / `error`

#### Просмотр файлов

**Панель файлов показывает:**
- Имя файла
- Размер файла
- Статус обработки:
  - `ready` — готов к воспроизведению
  - `processing` — обрабатывается
  - `error` — ошибка обработки
  - `checking` — проверяется
- Тип контента (видео, изображение, PDF, PPTX, папка)
- Дата загрузки
- **Наличие трейлера** (иконка)

**Пагинация:** Файлы отображаются постранично (по умолчанию 20 на страницу)

#### Превью файлов

**Для изображений:**
- Навести курсор на файл → показывается миниатюра
- Кликнуть на файл → открывается полноразмерное превью

**Для видео:**
- Кликнуть на файл → открывается видео плеер с трейлером (при наличии)
- Если трейлер ещё не сгенерирован — создаётся короткий клип через `/api/files/preview`

**Для PDF/PPTX:**
- Кликнуть на файл → открывается превью первой страницы
- Кнопки **"◀ Предыдущая"** и **"Следующая ▶"** для навигации

#### Переименование файла

1. Открыть устройство
2. В панели файлов найти нужный файл
3. Нажать на **"⋮"** (меню действий)
4. Выбрать **"Переименовать"**
5. Ввести новое имя
6. Нажать **"Сохранить"**

#### Удаление файла

1. Открыть устройство
2. В панели файлов найти нужный файл
3. Нажать на **"⋮"** (меню действий)
4. Выбрать **"Удалить"**
5. Подтвердить удаление

**⚠️ Внимание:** Файл удаляется с диска и из БД. Если файл используется другими устройствами (дедупликация), удаляется только ссылка.

#### Копирование файла между устройствами

1. Открыть устройство-источник
2. В панели файлов найти нужный файл
3. Нажать на **"⋮"** (меню действий)
4. Выбрать **"Копировать"**
5. Выбрать целевое устройство в списке
6. Нажать **"Копировать"**

**Особенности:**
- Используется дедупликация — если файл с таким MD5 уже есть, создаётся ссылка вместо копии

#### Установка файла по умолчанию (заглушка)

1. Открыть устройство
2. В панели файлов найти нужный файл
3. Нажать на **"⋮"** (меню действий)
4. Выбрать **"Сделать заглушкой"**

### Управление стримами

#### Добавление стрима
1. Открыть настройки устройства
2. Перейти в раздел **"Стримы"**
3. Ввести URL потока (HLS/DASH)
4. Нажать **"Добавить"**

**Поддерживаемые протоколы:**
- **HLS** — `.m3u8` плейлисты
- **DASH** — `.mpd` плейлисты

**Особенности:**
- Автотранскодинг для несовместимых кодеков
- Дедупликация — один стрим для нескольких устройств
- Circuit breaker — защита от падений источников
- Мониторинг — health check каждые 10 секунд

---

### Дедупликация файлов

**Цель:** Экономия места на диске за счёт использования одного физического файла для нескольких устройств.

**Как работает:**
1. При загрузке файла вычисляется MD5 хэш (частичный для файлов >100MB)
2. Система проверяет, есть ли файл с таким MD5 на других устройствах
3. Если есть — создаётся ссылка вместо копии
4. Файл физически хранится один раз

**Просмотр дубликатов:**
1. Нажать кнопку **"Устройства"** в toolbar
2. В модальном окне перейти на вкладку **"Дубликаты"**
3. Просмотреть список дубликатов

---

### Управление громкостью устройств

**Отображение:**
- В карточке устройства показывается иконка громкости и уровень (0-100%)
- В детальной информации устройства — ползунок громкости

**Управление:**
1. Открыть устройство
2. В детальной информации найти панель **"Громкость"**
3. Использовать ползунок для изменения уровня (шаг 5%)
4. Кнопка **"🔇"** / **"🔊"** для включения/выключения звука

---

### Управление пользователями

#### Создание пользователя
1. Нажать кнопку **"Пользователи"** в toolbar
2. Нажать **"Создать пользователя"**
3. Заполнить:
   - **Логин**
   - **Пароль**
   - **Роль** (admin, speaker, hero_admin)
   - **Назначенные устройства** (для speaker)
4. Нажать **"Сохранить"**

### Настройки системы

**Доступ:** Кнопка **"⚙️ Настройки"** в toolbar (только для admin)

**Настройки:**

1. **Хранилище контента:**
   - Путь к корневой директории данных (`contentRoot`)
   - Можно изменить на внешний диск

2. **Очистка базы данных:**
   - Проверка соответствия файлов в БД и на диске
   - Удаление записей о несуществующих файлах

3. **Ночная оптимизация:**
   - Настройка часов начала/конца
   - Профили кодирования (720p/1080p/2160p)
   - Лимиты ресурсов (CPU/memory)

4. **Self-update:**
   - Проверка обновлений из git
   - Применение обновлений

### Модули JavaScript

**Основной файл:** `public/js/admin.js`

**Модули (`public/js/admin/`):**
- `auth.js` — аутентификация (JWT токены)
- `devices-manager.js` — управление списком устройств
- `device-card.js` — отрисовка карточек устройств
- `device-crud.js` — CRUD операции с устройствами
- `files-manager.js` — управление списком файлов
- `file-actions.js` — действия с файлами (preview, rename, delete)
- `upload-manager.js` — загрузка файлов (Drag & Drop, кнопка)
- `upload-ui.js` — UI компоненты загрузки
- `socket-listeners.js` — обработчики Socket.IO событий
- `system-monitor.js` — мониторинг системы
- `ui-helpers.js` — вспомогательные функции UI
- `modal.js` — модальные окна
- `md5-helper.js` — MD5 дедупликация

### Socket.IO события

**Подписка на события:**
- `devices/updated` — обновление списка устройств
- `file/processing` — файл обрабатывается
- `file/progress` — прогресс обработки
- `file/ready` — файл готов
- `file/error` — ошибка обработки
- `file/cancelled` — задача оптимизации отменена
- `players/onlineSnapshot` — снимок онлайн устройств
- `devices/volume/state` — обновление громкости
- `devices/volume/stateBatch` — пакетное обновление громкости

### API Endpoints

**Основные endpoints:**
- `GET /api/devices` — список устройств
- `POST /api/devices` — создать устройство
- `DELETE /api/devices/:id` — удалить устройство
- `POST /api/devices/:id/upload` — загрузить файл
- `GET /api/devices/:id/files-with-status` — список файлов
- `POST /api/devices/:id/files/:name/rename` — переименовать файл
- `DELETE /api/devices/:id/files/:name` — удалить файл
- `POST /api/devices/:id/files/:name/copy` — копировать файл
- `GET /api/devices/:id/volume` — получить громкость
- `POST /api/devices/:id/volume` — установить громкость
- `GET /api/admin/settings` — получить настройки
- `POST /api/admin/settings/content-root` — изменить путь хранилища
- `GET /api/metrics` — метрики системы

## Решение проблем

### Файлы не загружаются

**Проверьте:**
1. Размер файла (максимум 5 GB)
2. Формат файла (должен быть в списке разрешённых)
3. Права доступа к папке устройства
4. Место на диске

### Устройство не отображается онлайн

**Проверьте:**
1. Устройство подключено к сети
2. Socket.IO соединение установлено
3. Device ID совпадает с настройками устройства

### Трейлер не генерируется

**Проверьте:**
1. FFmpeg установлен и доступен в PATH
2. Место на диске (кэш трейлеров в `data/cache/trailers/`)
3. Логи сервера: `journalctl -u videocontrol -f | grep trailer`
