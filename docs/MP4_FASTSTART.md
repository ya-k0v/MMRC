# MP4 Faststart - Перемотка в ExoPlayer

## Проблема

ExoPlayer не может перематывать MP4 файлы, у которых атом `moov` находится в конце файла. Это особенно критично для коротких клипов, где ExoPlayer не может найти метаданные без полной загрузки файла.

**Симптомы:**
- Длинные фильмы перематываются (ExoPlayer буферизует больше)
- Короткие клипы НЕ перематываются
- HTML5 плеер работает (более толерантен к структуре файла)

## Решение

Автоматическое применение `faststart` ко всем MP4 файлам при загрузке. Это перемещает атом `moov` и все необходимые таблицы в начало файла.

### Структура MP4 файла

**До обработки (не оптимизировано):**
```
[ftyp] [mdat] ... [moov]
```

**После обработки (оптимизировано):**
```
[ftyp] [moov] [mdat]
```

### Что переносится в moov атоме

Атом `moov` содержит ВСЕ необходимые данные для перемотки:

- **mvhd** (movie header) - общая информация о файле (длительность, временная шкала)
- **trak** (tracks) - треки видео/аудио с метаданными
  - **tkhd** (track header) - информация о треке
  - **mdia** (media) - медиа информация
    - **mdhd** (media header) - заголовок медиа
    - **hdlr** (handler) - тип трека
    - **minf** (media information) - информация о медиа
      - **stbl** (sample table) - **ТАБЛИЦЫ ДЛЯ ПЕРЕМОТКИ:**
        - **stco** (chunk offset) - смещения чанков
        - **stsc** (sample-to-chunk) - маппинг сэмплов к чанкам
        - **stsz** (sample size) - размеры сэмплов
        - **stts** (time-to-sample) - временные метки сэмплов
        - **stss** (sync sample) - ключевые кадры (I-frames)
        - **ctts** (composition time) - время композиции

Без `moov` в начале ExoPlayer не может:
- Определить длительность файла без полной загрузки
- Перематывать к произвольной позиции
- Найти ключевые кадры для перемотки

## Реализация

### Автоматическая обработка при загрузке

Все MP4 файлы (`.mp4`, `.m4v`, `.m4a`) автоматически обрабатываются при загрузке:

1. **Модуль:** `src/video/mp4-faststart.js`
   - `applyFaststart()` - применяет faststart к файлу
   - `needsFaststart()` - проверяет нужна ли обработка
   - `applyFaststartAsync()` - асинхронная обработка в фоне

2. **Интеграция:** `src/utils/file-metadata-processor.js`
   - Обработка запускается автоматически после сохранения метаданных
   - Выполняется в фоне, не блокирует ответ на загрузку
   - Пропускает файлы, которые уже оптимизированы

### Технические детали

**Используется команда:**
```bash
ffmpeg -i input.mp4 -c copy -movflags +faststart output.mp4
```

**Параметры:**
- `-c copy` - копируем все потоки БЕЗ перекодирования (быстро)
- `-movflags +faststart` - перемещает moov атом в начало

**Производительность:**
- Обработка происходит БЕЗ перекодирования
- Скорость зависит от размера файла и скорости диска
- Для файла 100MB обычно занимает 1-3 секунды

### Обработка существующих файлов

Для обработки уже загруженных файлов используйте утилиту:

```bash
# Обработать файлы конкретного устройства
node scripts/process-existing-mp4-faststart.js 001TV

# Обработать все файлы всех устройств
node scripts/process-existing-mp4-faststart.js --all
```

Утилита:
- Проверяет каждый файл перед обработкой
- Пропускает уже оптимизированные файлы
- Показывает прогресс и статистику

## Результат

После обработки:
- ✅ Все MP4 файлы можно перематывать в ExoPlayer
- ✅ Работает для файлов любого размера (короткие клипы и длинные фильмы)
- ✅ Быстрое начало воспроизведения (метаданные доступны сразу)
- ✅ Корректная работа перемотки через Range requests

## Проверка

Проверить что файл обработан можно через:

```bash
# Проверка структуры файла
ffprobe -v error -show_format input.mp4 | grep faststart

# Или проверка позиции moov атома
hexdump -C input.mp4 | head -20 | grep moov
```

Если `moov` найден в первых 100KB файла - файл оптимизирован.
